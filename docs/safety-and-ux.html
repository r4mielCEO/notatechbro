For developers

What it does—and what it does not do.

NotATechBro explains the likely impact of a command. It never decides whether the command should run.

The short version

It explains. You decide. Your coding agent still uses its normal approval controls.

Building an integration? The full wording rules are on GitHub.Open developer rules

It explains likely impact

rm -rf dist
→ This will delete the `dist` folder.
npm install
→ This will install or update this project's JavaScript packages.

It uses careful language

git clean -fd
→ This may permanently delete untracked files from the project.

When an explanation is approximate, NotATechBro says “may.” It never labels a command as safe.

It keeps the user in control

  • It does not run commands.
  • It does not approve or block commands.
  • It does not change the command.
  • It does not replace the agent's approval prompt.

It runs locally

Command explanations happen on the user's computer. NotATechBro makes no runtime cloud calls and sends no command text, file paths, or file contents to a server.

Short explanation. Honest limits. The user makes the decision.
