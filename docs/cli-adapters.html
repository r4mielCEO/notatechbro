CLI adapters

One preview engine. Different hook payloads.

Adapters normalize host-specific JSON into the same internal action model, then shared rules write the plain-English preview.

Supported today

  • Claude Code PreToolUse style payloads
  • Hermes pre_tool_call payloads
  • Codex-style shell/tool call candidates
  • Generic tool_name and tool_input JSON

Claude Code

Claude Code command hooks can pipe JSON into notatechbro. The older change-preview alias remains supported for existing configs. Human text goes to stderr by default so stdout stays available for host protocols.

{
  "tool_name": "Bash",
  "tool_input": { "command": "npm install" },
  "cwd": "/path/to/project"
}

Hermes

Hermes can call the CLI from a pre-tool hook. The tool stays observer-only and does not block.

{
  "hook_event_name": "pre_tool_call",
  "tool_name": "terminal",
  "tool_input": { "command": "git push" }
}

Codex-style payloads

Codex hook shapes vary by version, so this support is permissive until verified against the exact Codex CLI version you target.

{ "tool_call": { "name": "shell", "input": { "command": "pytest" } } }
Adapters are intentionally thin. Interpretation belongs in the shared local rule engine.
